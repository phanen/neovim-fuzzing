# Allowlist for AFL++ LTO instrumentation.
#
# Why allowlist vs denylist:
# nvim's binary is huge (~40MB). Instrumenting every translation unit
# blows up coverage map size and slows the dispatcher. allowlist
# keeps instrumentation focused on the surface that actually
# contains the bugs we want to find (C API + window + marktree +
# extmark). The LuaJIT side, the TUI, and the codegen helpers are
# excluded -- they cannot crash in our --headless fuzz path.
#
# Stability:
# The denylist (afl-denylist.txt) excludes a few specific files that
# would break the build (libnlua0.so symbol collisions). The allowlist
# is the opposite direction: only the files listed here get
# instrumented. When both are set, AFL++ prefers the allowlist (it
# overrides the default "instrument everything" behavior).
#
# Kevin Goodsell saw stability climb to 100% with a similar allowlist
# in 06-improvements of his fuzzing-nvim repo. Our allowlist is a
# superset covering all surfaces our fuzzers touch (win/buf/tab/
# autocmd/extmark/keys/etc).
